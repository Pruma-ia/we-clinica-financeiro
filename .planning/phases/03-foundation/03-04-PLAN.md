    Apply the seed by reading the file and executing it via the Neon serverless client (cannot use psql in all environments). Use node with --env-file:
    ```
    node --env-file=.env.local -e "
      const fs = require('fs');
      const sql = fs.readFileSync('drizzle/0001_seed.sql', 'utf8');
      import('@neondatabase/serverless').then(async ({neon}) => {
        const q = neon(process.env.DATABASE_URL);
        // Neon HTTP driver does not support multi-statement strings — split on semicolons.
        const statements = sql.split(/;\s*\n/).map(s => s.trim()).filter(s => s && !s.startsWith('--'));
        for (const stmt of statements) {
          await q(stmt);
        }
        console.log('SEED_OK');
      }).catch(e => { console.error(e.message); process.exit(1); });
    "
    ```

    The Neon HTTP driver does not support multi-statement strings in a single call, so the script splits the file on `;\n` and runs each statement separately. The three INSERTs are independent and idempotent.

    Note: comment lines (`-- ...`) at the top of the file are stripped by the `.filter(s => s && !s.startsWith('--'))` clause. Any inline comments inside an INSERT are kept by Postgres as-is.

    If the seed run fails mid-batch, the failure is logged. Because each INSERT uses `ON CONFLICT DO NOTHING` or `ON CONFLICT DO UPDATE`, re-running the entire seed is safe.
