| next/font → CDN at build | next/font fetches Google Fonts at build time (not runtime); offline build would fail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | Tailwind content globs | mitigate | Glob covers `./src/**/*.{js,jsx,ts,tsx,mdx}` — includes both new TS files and legacy `.jsx` so no utility class used in either is purged |
| T-04-01-02 | Information disclosure | next/font self-hosting | accept | next/font self-hosts fonts under `_next/static/media/` — no third-party CDN at runtime, no PII leakage via font requests; Google build-time fetch is acceptable for internal tool |
| T-04-01-03 | Denial of service | offline build | accept | If Vercel cannot reach fonts.googleapis.com at build, build fails — acceptable; Vercel has reliable egress, no offline build requirement for this MVP |
| T-04-01-SC | Tampering | npm/pnpm install (tailwindcss, postcss, autoprefixer) | accept | All three packages are official OpenJS Foundation / Tailwind Labs maintained, >1B downloads/year combined, no [SUS] or [SLOP] signal. No DISCOVERY.md package legitimacy table required — these are foundational web ecosystem packages |
